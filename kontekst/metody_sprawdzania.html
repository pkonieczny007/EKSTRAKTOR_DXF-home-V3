ZASADY NADRZĘDNE
  • Sprawdzanie < rysowanie od zera. Jeśli weryfikacja pozycji trwa dłużej niż narysowanie jej ręcznie — proces jest do wyrzucenia. Cel: zlecenie 20 pozycji sprawdzone w ~2 minuty.
  • Człowiek porównuje OBRAZKI, nie otwiera CAD-a. Otwarcie CAD-a do sprawdzenia = porażka procesu. CAD zostaje tylko do ręcznej poprawki odrzuconych.
  • Maszyna mówi jawnie, CZEGO jest niepewna. Żółty status zawsze z powodem („bbox OK, ale w widoku źródłowym były 2 okręgi, w wyniku 0"). Człowiek decyduje o jednej wątpliwości, nie szuka jej sam.

pomysł · opcja

Blink-diff A/B

szybkie przełączanie wynik/źródło w tym samym miejscu (jak w astronomii) — różnice „mrugają"; do trudnych przypadków, nie do codziennego przeglądu

Ergonomia przeglądu (żeby 20 pozycji = 2 minuty)

  • Kolejność: czerwone i żółte na górze galerii, zielone na dole — najpierw decyzje, potem kiwanie głową.
  • Batch approve: „zatwierdź wszystkie zielone" jednym klikiem po przewinięciu galerii.
  • Powód żółtego wypisany na kafelku — operator ocenia JEDNĄ wątpliwość, nie ogląda wszystkiego od nowa.
  • Klawiatura: Enter = zatwierdź, → = następny, X = odrzuć (z listą powodów do wyboru).
  • Zero nawigacji po folderach: galeria = jeden plik HTML per zlecenie, otwiera się z raportu.

Pętla uczenia — sprawdzanie karmi trening

  • każde odrzucenie/poprawka w galerii lub wyjątkach → wiersz w korpus/decyzje.csv (pozycja, kandydaci, wybór, POWÓD, cechy klastrów);
  • powody odrzuceń → statystyka → nowe bramki QC i progi (PLAN.md Etap 4) — system z każdym zleceniem pyta rzadziej;
  • fałszywe ZIELONE (wykryte w Lantku lub na produkcji) = najcenniejsza etykieta — natychmiast do testy/regresja.py jako nowy przypadek.

Kolejność wdrożenia w V2 (małe kroki, każdy domknięty)

1. bbox zrodlowy do CSV <- mala zmiana, odblokowuje ramke i overlay 2. galeria kafelkow (HTML) <- reuzycie _render_png.py + _demo_porownanie.py 3. weryfikator.py: bramki 3-6 <- filtr smieci, bilans otworow, izometria, giecia 4. semafor + batch approve <- statusy w galerii, decyzje.csv 5. interfejs wyjatkow (Flask) <- PLAN.md Etap 3, dopiero gdy 1-4 dzialaja

Po każdym kroku: python testy/regresja.py musi być PASS. V1 działa przez cały czas budowy.
