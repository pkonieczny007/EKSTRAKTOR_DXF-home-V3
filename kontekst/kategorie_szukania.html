Jak dodawać kategorie (architektura otwartej listy)

Każda kategoria = wtyczka o wspólnym interfejsie.
  • wejście: geometria (msp po rozbiciu bloków) + wiersz wykazu + profil klienta
  • wyjście: lista kandydatów, każdy z pewnością (0–1) i metrykami QC
  • rejestr kategorii w config/kategorie.yaml — kolejność, progi, włącz/wyłącz per profil klienta
  • orkiestrator odpala kategorie po kolei / równolegle i porównuje pewność — nie zna ich środka
  • dodanie kategorii = nowy plik w kategorie/ + wpis w YAML. Silnika nie dotykamy.
v2/ ├── orkiestrator.py # dobór + porównanie kandydatów ├── kategorie/ │ ├── __init__.py # rejestr (auto-discovery) │ ├── struktura.py # kat. 1 → woła funkcje z v1! │ ├── wymiary.py # kat. 2 │ ├── geometria.py # kat. 3 │ ├── adnotacje.py # kat. 4 │ ├── korpus.py # kat. 5 │ └── czlowiek.py # kat. 6 ├── weryfikator.py # wspólny QC dla wszystkich └── config/kategorie.yaml

Nie niszczymy obecnej pracy — projekt równoległy

Zasada: v1 zostaje nietknięty i działający przez cały czas budowy v2.
  • Nowy pakiet obok (np. src/v2/ albo folder EKSTRAKTOR_DXF_v2/) — v2 importuje sprawdzone funkcje z extract_positions.py (klastrowanie, bbox, zapis 1:1, wyśrodkowanie) zamiast je kopiować. Jedno źródło prawdy, zero dublowania bugfixów.
  • Stare CLI i skill /wyciagnij-dxf działają jak dotąd — v2 dostaje własne polecenie (np. python src/v2/orkiestrator.py ...).
  • Regresja podwójna: testy/regresja.py pilnuje v1 (musi być PASS zawsze); v2 dostaje benchmark v2 vs v1 na tych samych 3 rysunkach + realnych zleceniach — v2 „wygrywa" dopiero, gdy jest ≥ v1 na wszystkim.
  • Przełączenie: dopiero gdy v2 bije v1 na korpusie, skill przepinamy na v2. Do tego czasu v2 to laboratorium.

Notatki do przerysowania w Excalidraw

  • Układ: u góry pasek przepływu (szukanie → ekstrakcja → weryfikacja), pod nim 6 kolumn-kategorii + jedna pusta ramka „+" z prawej (przestrzeń na nowe).
  • Kolory ramek = kolory kategorii z tej strony (niebieski struktura, zielony wymiary, pomarańczowy geometria, fioletowy adnotacje, morski korpus, czerwony człowiek).
  • Status kartki: pełna linia = działa, przerywana = częściowo, szkicowa/szara = pomysł.
  • Strzałki: każda kategoria → wspólny WERYFIKATOR (jeden lejek — nic nie omija QC), z weryfikatora dwie strzałki: „pewne → Lantek" i „niepewne → kategoria 6 (człowiek) → korpus".
  • Ta strona to szkielet — jak dojdzie kategoria, dopisać kolumnę tu i na tablicy.
